# Notebook Run Order

Run the notebooks in numbered order — each step depends on the previous one.

## Pipeline

| # | Notebook | Purpose | Key Output |
|---|----------|---------|-----------|
| 01 | `01_feature_engineering.ipynb` | Convert raw `stock_txt/` → `stock_csv/` with technical indicators | `dataset/stock_csv/*.csv` |
| 02 | `02_pca_representation.ipynb` | Summarise per-stock statistics for clustering input | `../summary_statistics_2015_2017.csv` |
| 03 | `03_correlation_matrix.ipynb` | Sector-level correlation analysis | `dataset/sector/` |
| 04 | `04_kmeans_clustering.ipynb` | K-Means clustering → ranked pairs | `dataset/kmeans_pairs_ranked.csv` |
| 05 | `05_dbscan_clustering.ipynb` | DBSCAN clustering → ranked pairs | `dataset/dbscan_pairs_ranked.csv` |
| 06 | `spread_models/06_linear_regression.ipynb` | Pairs trading with fixed hedge ratio (LR) | MC plots, strategy vs BnH |
| 07 | `spread_models/07_kalman_filtering.ipynb` | Pairs trading with dynamic hedge ratio | Kalman spread results |

## Spread model module

All reusable logic for notebook 06 lives in `spread_models/linear_regression/`:

```
spread_models/linear_regression/
├── data_loader.py   load any pair from dataset/stock_csv/ by ticker
├── strategy.py      cointegration, hedge ratio, backtest, buy-and-hold
├── metrics.py       Sharpe, max-drawdown, win-rate
├── monte_carlo.py   MC simulation + 3σ outperformance test
└── run_pair.py      CLI: python run_pair.py --pairs_csv ../../dataset/kmeans_pairs_ranked.csv --top 5
```

## Prerequisites

Steps 04 and 05 require `summary_statistics_2015_2017.csv` (generated by step 02).
Steps 06 and 07 require `dataset/kmeans_pairs_ranked.csv` or `dataset/dbscan_pairs_ranked.csv` (generated by steps 04/05).
